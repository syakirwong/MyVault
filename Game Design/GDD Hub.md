# Stormbound Online — Game Design Document

## Vision

A **reactive text-based MMORPG** where build experimentation is the primary gameplay driver. Players interact through dynamic UI elements — buttons, panels, cards, and action logs — rather than typing commands or navigating pixel worlds. Depth comes from systems, gems, and skills, not graphics.

## Core Philosophy

- **Build Diversity First** — every meaningful action revolves around discovering or optimizing builds
- **Systems > Graphics** — skills, gems, dungeons, and economy provide gameplay depth
- **Reactive Interface** — players interact via UI buttons, dynamic panels, and action logs
- **No Pay-to-Win** — progression comes from dungeon runs, gem collection, and skill experimentation
- **Creator Ecosystem** — players discover, share, and optimize builds; builds define cosmetic identity and prestige

## Build Identity Balance

- Origin: **20%** of build identity (starting stats and passive ability)
- Skill Choice: **30%** (the canvas for gems)
- Gem Modifiers: **50%** (the main creativity driver)

# Gameplay Loops

## Primary Loop (The Main Event)

```
Prepare build
→ Queue / travel to dungeon
→ Clear dungeon (solo or party)
→ Obtain gems / gear / crafting materials
→ Modify build
→ Return to hub
→ Show build / trade / prepare next run
```

This loop aligns with the build experimentation philosophy. Players aren't grinding levels — they're grinding possibilities. Every dungeon run is a build test. Every gem drop is a new question: "Does this change my build?"

## Core Loop (Moment-to-Moment)

Combat → Loot Gems → Experiment with Builds → Show Off

The fundamental cycle players engage with every few minutes. Every combat encounter has the potential to drop gems that shift how a build feels. The "show off" step feeds into the [[Creator Ecosystem]] and social systems, making every build discovery a shareable moment.

## Session Loop (20-40 Minutes)

1. **Pick objective** — expedition dungeon, solo trial, farming run, PvP queue, or trade session
2. **Execute with current build** — test your gem configuration against real content
3. **Earn rewards** — gems, materials, currency, reputation
4. **Return to hub** — upgrade, experiment, trade at [[Social Hubs]]
5. **Share results** — publish build updates or trade discoveries

Each session should feel complete. A player who logs in for 30 minutes should walk away having made meaningful progress or discovered something new about their build.

## Weekly Loop

- **Mythic dungeon rotation** — new Mythic dungeon each week creates community-wide hype and meta discussion
- **Challenge Mode modifiers** — weekly dungeon modifier rotations shift the meta, forcing build adaptation (see [[PvE Structure]])
- **Solo Trial rotation** — weekly trial tests different combat axes (burst, sustain, mobility)
- **PvP season progression** — weekly point thresholds and ranked milestones (see [[PvP Structure]])
- **Trade route shifts** — supply and demand changes across regions, creating new arbitrage opportunities (see [[Trade System]])
- **Community build challenges** — themed constraints that reward creative solutions (see [[Creator Ecosystem]])

## Meta Loop (Long-Term)

- **Build mastery and reputation** — become known for specific playstyles or innovations (see [[Progression Systems]])
- **Creator leaderboard climbing** — publish builds, earn followers, gain recognition
- **Guild progression and territory** — collective goals that require coordinated builds
- **Alignment progression and story arcs** — deep narrative choices that shape available gems and visuals (see [[Alignment System]])
- **Collection completion** — gem encyclopedia, cosmetic sets, achievement milestones

## Between-Dungeon Activities

These systems exist to support the dungeon loop, not compete with it. They feed resources, context, or social value back into the primary progression path.

### PvP

| Reward Type | Examples | Design Intent |
|---|---|---|
| PvP tokens | Spend on rare modifiers, cosmetics | Alternate acquisition path, not primary |
| Cosmetic unlocks | Ranked auras, titles, seasonal sets | Prestige without power advantage |
| Alignment points | Alignment progression from PvP events | Feeds into [[Alignment System]] depth |

PvP does NOT drop primary gear or expedition-tier gems. PvE players should never feel forced into PvP for progression. PvP rewards are lateral — cosmetic, prestige, and alternate modifier access.

### Trade Routes

Players escort cargo between ports or hubs.

| Reward Type | Examples |
|---|---|
| Crafting materials | Bulk materials for [[Crafting System]] recipes |
| Gold | Currency for market transactions |
| Economy resources | Region-specific reagents tied to [[Trade System]] supply |

Trade routes feed the crafting market. Players who enjoy logistics and economy have a path that directly supports the dungeon-running population.

### Guild Wars

Large-scale organized events.

| Reward Type | Examples |
|---|---|
| Territory buffs | Minor regional bonuses (XP, drop luck) for controlling guilds |
| Tax bonuses | Small gold income from controlled trade hubs |
| Guild prestige | Leaderboard ranking, cosmetic guild hall upgrades |

**No direct power advantages.** Territory buffs are minor and convenience-oriented. Snowballing guild dominance through power advantages is a design failure — dominance should come from coordination and skill, rewarded with prestige and cosmetics.

### Open World

| Activity | Purpose |
|---|---|
| Exploration | Hidden mini-dungeons, treasure maps, lore |
| Resource gathering | Crafting nodes, rare spawns |
| World bosses | Community events, guaranteed contribution rewards |
| Daily bounties | Short repeatable content (15-20 min) |

The open world is the connective tissue. It provides variety and breathing room between dungeon runs, but it is not the progression engine.

## Engagement Drivers

| Driver | System | Reward Type |
|---|---|---|
| Mastery | [[Gem System]], Build optimization, [[Combat System]] | Power + Visual |
| Social | Guilds, Build sharing, [[Trade System]] | Recognition + Currency |
| Collection | Gem encyclopedia, [[Cosmetic System]] | Completion + Visual |
| Competition | [[PvP Structure]], Solo Trials, Leaderboards | Rank + Cosmetics |
| Expression | Build creation, [[Alignment System]] | Identity + Recognition |

## Anti-Churn Mechanics

- **Daily login bonus** — crafting material or random gem (small, never mandatory)
- **Session pacing** — meaningful rewards within 20-40 minutes, no content that demands multi-hour sessions to feel worthwhile
- **No FOMO** — seasonal content remains accessible permanently; only cosmetic exclusives rotate on a schedule
- **Mythic rotation, not removal** — Mythic dungeons rotate weekly but cycle back. Nothing is permanently missable

The goal is to make players WANT to log in, not feel OBLIGATED to. Respect for player time is a core design value.
